1. **Datos generales del Sprint 1**

Fecha: 15 de diciembre de 2025  
Duración del Sprint: 1 semana (del 16 al 22 de diciembre)  
Participantes: Bryan, Javier, Izan, Carlos  
Scrum Master: Bryan

2. **Objetivo del Sprint**

Desplegar la aplicación Extagram en una sola máquina con:  
\- Servidor web funcional (Apache)  
\- Base de datos MySQL operativa  
\- Sistema de subida de imágenes funcionando  
\- Código subido a GitHub  
\- Documentación inicial completa

3. **Requisitos seleccionados**  
     
   1.Instalación y configuración de Apache  
   2\. Instalación de PHP-FPM  
   3\. Instalación de MySQL  
   4\. Creación de base de datos y tabla posts  
   5\. Despliegue de extagram.php  
   6\. Implementación de upload.php  
   7\. Creación del repositorio GitHub  
   8\. Documentación técnica inicial  
     
4. **Tareas del Sprint**

| TÍTULO | DESCRIPCIÓN |
| :---- | :---- |
| P1 \- Preparación del sistema base | Configurar la máquina virtual o servidor con el sistema operativo necesario. |
| P2 \- Instalación del servidor web | Instalar y configurar NGINX o Apache para servir la aplicación web. |
| P3 \- Instalación de PHP | Instalar PHP y los módulos necesarios para ejecutar correctamente los scripts PHP. |
| P4 \- Configuración de PHP | Configurar PHP para integrarlo con el servidor web y permitir la ejecución de código PHP. |
| P5 \- Despliegue de extagram.php | Colocar el archivo extagram.php en el servidor web y verificar su funcionamiento a través del navegador. |
| P6 \- Instalación de MySQL | Instalar el servidor de base de datos MySQL o MariaDB. |
| P7 \- Creación de base de datos | Crear la base de datos extagram\_db. |
| P8 \- Creación de usuario MySQL | Crear el usuario extagram\_admin y asignarle los permisos correspondientes sobre la base de datos. |
| P9 \- Creación de tabla posts | Crear la tabla posts con los campos necesarios según el esquema del proyecto. |
| P10 \- Conexión PHP–MySQL | Configurar y comprobar la conexión entre el script PHP y la base de datos. |
| P11 \- Implementación de upload.php | Configurar el script upload.php para permitir la subida de imágenes. |
| P12 \- Creación del directorio uploads | Crear la carpeta donde se almacenarán las imágenes subidas. |
| P13 \- Configuración de permisos | Asignar permisos de escritura adecuados en la carpeta uploads. |
| P14 \- Pruebas de subida de imágenes | Comprobar que las imágenes se suben y se guardan correctamente. |
| P15 \- Visualización de imágenes | Verificar que las imágenes subidas se muestran correctamente en la web. |
| P16 \- Creación del repositorio GitHub | Crear el repositorio del proyecto en GitHub para el control de versiones. |
| P17 \- Subida del código al repositorio | Subir el código fuente y los archivos iniciales al repositorio usando Git. |
| P18 \- Estructura de documentación | Crear la estructura de carpetas y archivos Markdown para la documentación. |
| P19 \- Documentación técnica | Documentar el análisis, la instalación, la base de datos, la subida de imágenes y cualquier configuración realizada. |
| P20 \- Acta de Sprint Review | Redactar el acta de revisión del sprint en Markdown y subirla al repositorio. |
